

# מבנה היררכי לסוגי הדרכות – תוכנית מימוש

## מה נבנה
מבנה דו-שכבתי: **קטגוריות** (עבודה בגובה, הדרכה כללית...) → **תתי-נושאים** (סולמות, פיגומים...), עם תצוגת אקורדיון ו-CRUD לשתי הרמות.

## שינויים

### 1. מודל נתונים (`src/data/mockData.ts`)
- הוספת `TrainingCategory` interface (id, name, description)
- הוספת `categoryId` ל-`TrainingType`
- נתוני דוגמה: 4 קטגוריות (עבודה בגובה, הדרכה כללית, חומרים מסוכנים, מלגזה) עם תתי-נושאים

### 2. דיאלוגים חדשים
- **`TrainingCategoryDialog.tsx`** – טופס הוספה/עריכת קטגוריה (שם + תיאור)
- **`TrainingTypeDialog.tsx`** – טופס הוספה/עריכת תת-נושא (שם, תוקף, תחום, דורש תעודה, קטגוריה)

### 3. מסך סוגי הדרכות (`src/pages/TrainingTypes.tsx`)
- שכתוב לתצוגת Accordion – כל קטגוריה נפתחת ומציגה טבלת תתי-נושאים
- כפתור "קטגוריה חדשה" בראש העמוד
- כפתור "תת-נושא חדש" בתוך כל קטגוריה
- עריכה ומחיקה בשתי הרמות

### 4. עדכון plan.md
- תיעוד המבנה החדש

## פרטים טכניים
- שימוש ב-Accordion הקיים בפרויקט (`@radix-ui/react-accordion`)
- הדרכות פשוטות = קטגוריה עם תת-נושא יחיד
- helper function חדש: `getCategoryName(categoryId)`
